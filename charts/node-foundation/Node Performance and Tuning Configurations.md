# **Node Performance and Tuning Configurations**

This document provides a structured breakdown of the hardware performance tuning settings based on the provided specifications. It encompasses the kubelet PerformanceProfile, Tuned sysctls, kernel boot arguments, and CRI-O memory limits designed to ensure low-latency, dedicated performance for complex workloads.

## **1\. Performance Profile Settings**

The table below details the expected kubelet configurations generated by the Node Tuning Operator (NTO), establishing the core CPU, memory, and networking reservation policies.

| Parameter | Configured Value | Description (What) | Rationale (Why) | Impact if Misconfigured   |
| :---- | :---- | :---- | :---- | :---- |
| **cpu.reserved** | 0-7,72-79 | Defines the dedicated CPU cores allocated for OpenShift system services, such as CRI-O, Kubelet, NIC IRQ and other essential infrastructure daemons. | Allocates 8 cores per socket. When data arrives from the network, it must be handled by the workers on that exact same chip to avoid delay.  | Rail IRQs land on isolated cores (causing jitter) or cross sockets (latency). |
| **cpu.isolated** | 8-71,80-143 | The exclusive pool handed by kubelet's static CPU manager to Guaranteed pods. | The exact complement of reserved CPUs. 128 CPUs matches the integer request of one full-node GPU worker pod. | Overlap with reserved cores is rejected by NTO. A gap wastes cores silently. |
| **numa.topologyPolicy** | best-effort | Dictates how strictly kubelet insists a pod's CPUs, memory, and devices come from one NUMA node. | Worker pods are full-node by design (spanning both sockets). Stricter policies like single-numa-node would permanently reject the pods at admission. | Stricter settings cause GPU pods to be permanently stuck in Pending/TopologyAffinityError. |
| **hugepages** | 16 × 1G | Boot-time-reserved 1 GiB pages usable by pods. | The Hugepages allocation is intentionally restricted to a 16 GB buffer. The KV-cache tier relies on CUDA pinned memory rather than hugetlbfs; therefore, over-allocating Hugepages would permanently consume standard RAM and degrade overall system availability. | If too big, the CPU cache tier ordering breaks; if too small, hugetlbfs consumers fail to map. |
| **globallyDisableIrqLoadBalancing** | false | Maintains normal IRQ balancing node-wide while letting individual pods opt their exclusive CPUs out. | This setting is half of our strategy for achieving ultra-low latency. Instead of disabling system interrupts for the entire server, we leave them enabled. Then, we use special labels on our heavy-duty AI pods that tell the system to route all background noise away from them. Regular pods run normally, while VIP pods stay strictly isolated. | If true, reserved cores are starved by all IRQ load. If false without annotations, isolation silently degrades. |
| **net.userLevelNetworking** | false | Prevents sizing NIC queues down to the reserved CPU count. | RDMA relies on kernel-bypass but is not DPDK. The rails require their full queue spread to function optimally. | NIC queue bottlenecks if erroneously enabled. The system will limit the NIC queues according to the 8 reserved CPU. |
| **workloadHints** | realTime: true highPowerConsumption: false perPodPowerManagement: false | Selectors that tell NTO which tuning bundles to activate in the generated profile. | These are the NTO defaults, but we pin them explicitly: realTime: true is what makes NTO generate most of the classic low-latency kernel args (nohz\_full, nosoftlockup, skew\_tick, tsc=reliable...) for us, so we don't have to list them ourselves. | Flipping realTime to false silently drops seven latency args on the next render — nothing errors, the node just gets slower and jittery. |

## **2\. Tuned Profile, Sysctls & System Configurations**

The following table outlines the Tuned child profile system controls (sysctls), essential kernel boot arguments, and runtime limits configured to support large-scale direct memory access, networking, and latency reduction.

| Configuration Parameter | Target Value | Default Value | Purpose & Impact   |
| :---- | :---- | :---- | :---- |
| **vm.swappiness** | 0 | 60 (NTO profile lowers it to 10) | Never swap while any file cache remains. With \~1 TB of CUDA-pinned memory, swapping anything the inference stack touches means multi-ms stalls; 0 means "only when the alternative is OOM". |
| **vm.zone\_reclaim\_mode** | 0 | 0 (Pinned) | Forces a NUMA node out of free pages to borrow from another node instead of stalling for local reclaim, reducing jitter. |
| **vm.max\_map\_count** | 1048576 | 65,530 | Accommodates the thousands of mmap'd weight shards and CUDA allocations required by large AI models without triggering cryptic ENOMEM errors. |
| **net.core.rmem\_max / wmem\_max** | 536870912 | \~208 KiB | Provides a 512 MiB ceiling for socket buffers serving TCP paths, allowing bulk staging transfers on 100/400G links to fill the pipe. |
| **net.ipv4.tcp\_rmem / tcp\_wmem** | 4096 87380 268435456 | Standard | Raises the TCP autotune maximum to 256 MiB for long-fat staging transfers without ballooning memory on idle connections. |
| **fs.aio-max-nr** | 1048576 | 65,536 | Expands the max in-flight async I/O requests to support the deep async queues of the NVMe KV-cache disk tier, avoiding EAGAIN errors. |
| **vm.min\_free\_kbytes** | 4194304 | Scale-dependent | Creates a 4 GiB free-page reserve preventing atomic allocation failures under pressure on RDMA-heavy hosts. |
| **ARP-Flux Suppression** | arp\_ignore=1 arp\_announce=2 arp\_filter=1 | 0 / 0 / 0 | Hardens L2 networking by ensuring each interface explicitly handles only its own configured traffic, preventing cross-rail flux on multi-homed hosts. |
| **transparent\_hugepages** | madvise | always (NTO profile sets never) | Restricts THP to applications that explicitly opt-in (CUDA/PyTorch allocators do), mitigating background khugepaged compaction stalls. A deliberate override of the NTO profile's blanket never. |
| **Kernel: Chart-set args (additionalKernelArgs)** | numa\_balancing=disable pcie\_aspm=off rcu\_nocb\_poll intel\_idle.max\_cstate=1 processor.max\_cstate=1 | Enabled / Auto | Only the args NTO does not generate: disables auto-NUMA page migration (fights static pinning), disables PCIe link power states (millisecond wakeup penalties on GPUDirect RDMA/NVMe), makes offloaded RCU threads poll instead of interrupting isolated cores, and caps CPU sleep at C1 so cores never pay deep-sleep wakeup latency mid-decode. |
| **Kernel: NTO auto-generated (never re-add by hand)** | isolcpus, nohz\_full, rcu\_nocbs, systemd.cpu\_affinity, hugepages args, nosoftlockup, skew\_tick=1, rcutree.kthread\_prio=11, tsc=reliable, nmi\_watchdog=0, mce=off, intel\_iommu=on iommu=pt, intel\_pstate | Generated by NTO | NTO builds these from the cpusets, the realTime workload hint, and the CPU-vendor include: CPU isolation and tickless cores, watchdog/MCE-polling noise off, a trustworthy TSC clock, and IOMMU passthrough for SR-IOV \+ GPUDirect RDMA. Re-adding any of them manually creates duplicate args that silently drift when the cpusets change. |
| **CRI-O: default\_ulimits** | memlock=-1:-1 | Tens of MiB | Removes the locked-memory rlimit globally, allowing RDMA/NCCL to safely pin the gigabytes of physical memory required without crashing the pod on startup. |
